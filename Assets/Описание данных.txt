Создать приложение для тестов, у него будет разные
источники данных, как внешние файлы, сгенерированные
данные, так и данные полученные из сети. Эти данные
будут вопросами тестов. Тесты должны отображаться 
на экране, при этом у каждого теста должно быть свое 
представление на экране. В тестах может содержаться
как один, так и множество вопросов. За каждый вопрос
теста назначается одинаковое кол-во очков. Некоторые
тесты должны быть на время. При этом итоговое кол-во 
очков может зависить от времени за которое тест был
пройден. У каждого теста будет свой уровень сложности
он определяет кол-во очков, которые пользователь
может получить или потерять за один вопрос. Если 
пользователь делает одну ошибку то отнимается половина
от максимального кол-ва балов, если две, то пользователь
ничего не получает, если три ошибки то минус половина
от максимального кол-ва и т.д.

Система будет состоять из следующих частей:
Model
Часть модель данных, куда будут загружаться данные
вопросов.
Модель данных должна подходить для каждого типа тестов.
Поскольку тесты отличаются типами вопросов и ответов
это нужно учесть.

View
Часть представления данных. Отображение того, как 
данные должны располагаться на окнах. 
Представления вопросов и тестов должны формироваться
на основе моделей вопросов и тестов. Т.е. эти 
представления должны получать конкретные объекты 
моделей вопросов и формировать конкретные визуальные
компоненты на основе этих вопросов. Таким образом 
должны быть сделаны представления для одного 
конкретного вопроса. После чего на основе этих 
представлений одного вопроса должен подключится 
функционал теста.
